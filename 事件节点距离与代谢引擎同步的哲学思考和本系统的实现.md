# 事件节点距离与代谢引擎同步的哲学思考和本系统的实现

## 1. 哲学思考：时间是良药，遗忘把无关记忆推远

在人类的认知拓扑中，记忆并不是永恒不变的刻录，而是一个随时间不断代谢、风化、重塑的动态过程。

### 1.1 记忆的模糊与距离的拉远
“时间是良药，都会过去的。”这句话在因果星空中有着精确的数学映射。当一个事件发生时，它充满了丰富的细节（全息文本），在我们的脑海中占据着核心位置。但随着时间的流逝，如果不去反复回想（缺乏因果连线和权重激活），这段记忆就会被“代谢”。
代谢的过程，就是细节被剥离、内容被提炼的过程。原本生动具体的记忆，变成了一个模糊的概括，甚至最终变成一个空壳。在这个过程中，这段记忆与其他事物的**关联性（相似度）必然会下降**，它在我们的心理空间中被推得越来越远。

### 1.2 潜意识（向量）与表层意识（文本）的统一
在早期的设计中，我们为了保证检索的绝对精准，让节点的语义向量（潜意识）保持不变，只让文本内容（表层意识）随时间代谢。
但这导致了一个“身心不一”的认知错位：对于 Agent 来说，当它进入事件视界时，它会发现一些文本已经被代谢得面目全非的节点，依然凭借着最初全息状态下生成的锐利向量，死死地钉在视界中心。这违背了人类遗忘的自然规律。

真正的遗忘，不仅是表层文本的缩减，更是底层潜意识特征的泛化与丢失。

## 2. 本系统的实现方案：第一性原理的胜利

为了完美模拟人类的遗忘机制，我们放弃了在距离公式中生硬地加入“时间惩罚常量”的工程捷径，而是选择回归第一性原理：**让向量同步跟随节点内容代谢而更新。**

### 2.1 坚持纯粹的语义距离算法
我们保持事件视界的距离算法绝对纯粹，不受任何外部变量的直接干预：
`距离 D = (1 - 余弦相似度) * 100`
一个节点是否在视界内（`D <= MAX_EYES`），完全取决于它当前的语义向量与聚焦节点的语义向量之间的真实数学夹角。

### 2.2 向量同步代谢（遗忘的推远）
当代谢引擎（`metabolism.py`）因为时间流逝和权重下降，将一个节点的详细内容提炼为简短摘要（甚至清空）时，系统会**同步为这个提炼后的新文本重新计算语义向量**。
由于新文本丢失了大量细节特征，新生成的向量在多维语义空间中会发生偏移，变得更加“泛化”。这种泛化直接导致它与当前聚焦节点的余弦相似度下降，计算出的距离 $D$ 变大。
当 $D$ 超过 `MAX_EYES` 时，这个节点就会自然而然地跌出事件视界。Agent 感受到的，将是一个真正随着时间流逝而变得模糊、疏离的记忆网络。

### 2.3 地宫唤醒与锐化（记忆的拉回）
遗忘并非彻底的删除。当一个被推远的节点因为某种强烈的刺激被重新激活时，系统会从地宫（Necropolis）中恢复它的全息文本。
此时，系统会**再次同步为全息文本重新计算语义向量**。随着细节的回归，向量重新“锐化”，它在语义空间中的引力瞬间恢复。距离 $D$ 迅速缩小，这个节点又会被重新拉回事件视界，如同尘封的记忆被瞬间唤醒，历历在目。

---

**总结**：通过让向量字段与节点内容保持绝对的同步代谢，我们无需修改任何距离判定公式，就利用大语言模型 Embedding 本身的降维特性，完美实现了“时间遗忘把无关节点推远”的哲学构想。这不仅在视觉上照顾了关系，更在 Agent 的底层认知逻辑上实现了真正的拟人化。从而实现“因果是多意志主体相互影响下的多因多果的引力场”，用四维时空实现对它的事件物理映射。
