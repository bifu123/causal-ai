# 用向量距离取代原生3D距离

## 现状与原因分析

在之前的 3D 星空中，节点“距离近的反而很远，即使距离近也会分散”的根本原因在于 D3 力导向图（Force Graph）的参数配置不合理，导致原生 3D 物理引擎的排斥力掩盖了我们计算出的向量距离：

1. **排斥力过大**：全局的电荷排斥力（`charge`）被设置为了 `-200`，这会导致所有节点无差别地强烈相互推开。
2. **因果连线拉力极弱**：对于因果关系连线（`type === 'causal'`），其拉力强度（`strength`）仅为 `0.02`，而距离被固定在 `250`。这意味着因果连线几乎无法将节点拉近，完全被 `-200` 的全局排斥力所冲散。
3. **语义连线拉力不均**：语义连线的拉力虽然最高可达 `1.5`，但最低只有 `0.001`，同样容易被排斥力冲散。

## 修改方案

为了让“距离成为节点之间视觉渲染的唯一考量标准”，我们需要让**连线（Link）的约束力占据绝对主导地位**，并削弱无差别的全局排斥力。

### 修改文件
`static/js/3d_main.js`

### 具体修改点

#### 1. 削弱全局排斥力
**位置**：约第 2556 行（`Graph.d3Force('charge')` 配置处）
**修改前**：
```javascript
Graph.d3Force('charge').strength(-200); 
```
**修改后**：
```javascript
Graph.d3Force('charge').strength(-50); 
```
**原因**：将排斥力从 `-200` 降低到 `-50`，防止节点被无差别地炸开，为连线拉力发挥作用提供基础。

#### 2. 增强因果连线的拉力与调整距离
**位置**：约第 2588 行（`const linkForce = Graph.d3Force('link');` 配置处）
**修改前**：
```javascript
.distance(link => {
    if (link.type === 'semantic') {
        // ...
    } else {
        return 250; // 因果连线距离
    }
})
.strength(link => {
    if (link.type === 'semantic') {
        // ...
    } else {
        return 0.02; // 因果连线拉力
    }
});
```
**修改后**：
```javascript
.distance(link => {
    if (link.type === 'semantic') {
        // ...
    } else {
        return 150; // 缩短因果连线距离
    }
})
.strength(link => {
    if (link.type === 'semantic') {
        // ...
    } else {
        return 1.0; // 大幅增强因果连线拉力
    }
});
```
**原因**：将因果连线的 `distance` 从 `250` 缩短至 `150`，并将 `strength` 从 `0.02` 提升至 `1.0`（强约束），让有直接因果关系的节点能够紧密靠拢。

#### 3. 统一语义连线的强约束
**位置**：同上（`linkForce` 的 `strength` 回调函数中）
**修改前**：
```javascript
// sim = 1.0 -> str = 1.5 (极强拉力)
// sim = 0.6 -> str = 0.001 (极弱拉力)
const str = Math.max(0.001, Math.min(1.5, 0.001 + (sim - 0.6) * (1.499 / 0.4)));
```
**修改后**：
```javascript
// 提高语义连线的拉力下限，确保距离成为唯一标准
// sim = 1.0 -> str = 1.5 (极强拉力)
// sim = 0.6 -> str = 0.5 (较强拉力)
const str = Math.max(0.5, Math.min(1.5, 0.5 + (sim - 0.6) * (1.0 / 0.4)));
```
**原因**：调整语义连线的 `strength`，使其下限也保持在一个合理的约束范围内（`0.5` 到 `1.5`），确保只要有连线，就能严格按照计算出的 `distance` 保持距离，而不会被排斥力推开。
