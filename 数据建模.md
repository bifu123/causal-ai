## 一、 人类健忘模型在因果链数据设计的启示

人类的记忆系统是一个极其精妙的**"资源受限下的高可用系统"**。将其映射到因果AI的数据架构中，我们得到以下核心启示：

1. **认知经济性与存储降级 (Cognitive Economy)**
   * **启示**：大脑无法、也不应存储所有全量像素和逐字对话。系统带宽必须留给"当前最关心的因果"。
   * **映射**：引入"生存权重 (Survival Weight)"。随着时间推移，无激活的节点将失去存储配额，从"全量记录"降级为"摘要"，最后只剩下一个"事件二元组"。
2. **抽象化即智慧 (Abstraction is Wisdom)**
   * **启示**：当你忘记了"张三昨天用左手还是右手打了李四"的细节，你只记住了"张三有暴力倾向"。细节的遗忘，促成了规律（果）的诞生。
   * **映射**：节点被压缩到极限时，剩下的 `event_tuple`（如：商王-求-人牲）就是放之四海而皆准的逻辑骨架。
3. **视觉的内外分离 (Visual Tiered Degradation)**
   * **启示**：对于远期记忆，人类脑海中只有一张模糊的、低分辨率的"缩略图"作为情绪和逻辑的锚点。若想看清细节，需要回到案发地（外部检索）。
   * **映射**：将"全量高清图"作为外显子（存放于外部OSS/URL），将"低清缩略图"作为内显子（直接存入数据库）。当生存权重降低时，系统会毫不犹豫地斩断外部URL，仅靠体内的缩略图维持视觉感应。

## 二、 渐忘与地宫迁移算法 (Python)
该算法实现了"像素-权重耦合"机制，确保 AI 的逻辑记忆与视觉感知的精度同步衰减。

```python
import math
import time
import os
from uuid import uuid4

class NecropolisMetabolismEngine:
    """
    代谢自愈引擎
    职责：驱动权重衰减，执行“灵魂出窍（地宫封存）”，控制内容精炼与视觉压缩。
    """
    def __init__(self, db, llm, vision_tool):
        self.db = db
        self.llm = llm            # 职责：提炼事件二元组
        self.vision = vision_tool # 职责：重采样像素
        self.lambda_const = 0.005 # 衰减常数
        self.event_max = int(os.getenv("EVENT_MAX", 200))

    def process_node(self, node):
        # 1. 计算当前权重
        dt_hours = (time.time() - node['last_accessed'].timestamp()) / 3600
        w = round(math.exp(-self.lambda_const * dt_hours), 4)
        node['survival_weight'] = w

        # 2. 地宫封存业务逻辑 (肉身沉降)
        # ---------------------------------------------------------
        # 条件：内容长度超过 EVENT_MAX 且 权重开始下降 (W < 0.95)
        if len(node['event_tuple'] or "") > self.event_max and w < 0.95 and node['compression_level'] == 0:
            # A. 开启地宫：封存原始肉身
            nec_id = uuid4()
            self.db.insert_necropolis(
                necropolis_id=nec_id,
                node_id=node['node_id'],
                raw_content=node['event_tuple'], # 封存全量原文
                raw_full_image_url=node['full_image_url']
            )
            # B. 意志提炼：LLM 提炼为带时间元素的二元组 (什么时间|谁做了什么)
            # 要求：尽量详细但严控在 EVENT_MAX 之内
            refined_tuple = self.llm.extract_event_tuple(
                text=node['event_tuple'], 
                max_len=self.event_max
            )
            
            # C. 活跃层卸载
            node['necropolis_id'] = nec_id
            node['event_tuple'] = refined_tuple
            node['full_image_url'] = None # 物理断开高清路径
            
            # D. 视觉同步降维 (1024 -> 256)
            node['thumb_image_blob'] = self.vision.resample(node['thumb_image_blob'], res=256)
            node['current_pixel_res'] = 256
            node['compression_level'] = 1
            node['vision_level'] = 1

        # 3. 深度渐忘业务逻辑 (化桩)
        # ---------------------------------------------------------
        elif w < 0.2 and node['compression_level'] == 1:
            # 视觉降维至像素锚点 (8px)
            node['thumb_image_blob'] = self.vision.resample(node['thumb_image_blob'], res=8)
            node['current_pixel_res'] = 8
            node['compression_level'] = 2
            node['vision_level'] = 2

        # 4. 湮灭业务逻辑 (W归零)
        # ---------------------------------------------------------
        # 注意：保持记录存在，仅清空意志内容
        elif w <= 0 and node['compression_level'] < 3:
            node['event_tuple'] = None      # 意志湮灭，设为NULL
            node['thumb_image_blob'] = None # 视觉彻底湮灭
            node['current_pixel_res'] = 0
            node['compression_level'] = 3
            node['vision_level'] = 3

        return node
```

## 双轨检索与全息重构 (The Retrieval & Reconstruction Protocol)

**核心理念：** 检索不仅是寻找数据，而是为了**"唤醒重构"**。系统首先通过轻量级的活跃表进行感应，一旦产生高强度对贞，则跨级开启地宫进行全息重构。

---

### 模式一：逻辑感应检索 (RAG & Causal Topology)

这是基于文本语义与因果拓扑的检索路径。它模拟人类通过"关键词"和"逻辑链"去回溯事件的过程。

#### 1. 检索流 (The Search Flow)
* **第一阶段：RAG 语义锚定**
    * 通过 `semantic_vector` 在活跃表中进行 HNSW 向量搜索。
    * 定位到最相似的 `event_tuple` 节点（此时可能是摘要、二元组或桩）。
* **第二阶段：拓扑扩散 (Parent-Child Search Tool)**
    * 调用系统内置的 `node_traversal_tool`。
    * 根据 `parent_id` 和 `serial_id` 向上追溯"前因"，向下探测"后果"。
    * 横向搜索"对贞"节点，还原整个因果块（Block）的逻辑全貌。

#### 2. 职责分工 (Responsibilities)
* **语义指纹**：负责在千万级节点中毫秒级定位。
* **父子ID工具**：负责在定位点周围"织网"，防止碎片化记忆。

---

### 模式二：视觉直觉检索 (Visual Intuition & Holographic)

这是模拟人类"似曾相识"（Déjà Vu）的检索路径。它不依赖文字，直接通过视觉特征穿透渐忘层。



#### 1. 检索流 (The Search Flow)
* **第一阶段：特征对贞 (Visual Peering)**
    * 系统提取输入图片的 `visual_vector`（视觉特征指纹）。
    * 在活跃表中搜索匹配的视觉残留。即使该节点已压缩至 8px 分辨率，只要特征指纹匹配，即可产生震荡。
* **第二阶段：越级调取 (Necropolis Leap)**
    * 一旦视觉相似度超过阈值（如 0.92），系统跳过活跃表的文字笔记。
    * 根据 `necropolis_id` 越级开启地宫，拉取 `raw_full_image_url` 和 `holographic_bundle`。

#### 2. 由图定事 (Image-to-Event Determination)
* **全息重构**：从地宫中提取全景图及其关联的原始 `raw_content`。
* **因果定性**：由系统判定："我看到了这张图中的特定祭坛裂纹（视觉锚点），对照地宫档案，确定这是‘甲寅年地震’期间的求祭事件。"

---

### 三、 检索结果的"意志响应"

当两种检索路径给出结果后，系统需进行权限与成本评估：

| 动作 | 描述 | 消耗 |
| :--- | :--- | :--- |
| **浅层回应** | 仅输出活跃表的 `event_tuple` (摘要/笔记) | 极低 |
| **深层重构** | 开启地宫，输出 `raw_content` 和全息图 | 高 (需 Actor 授权) |

> **设计原则：** > 检索不是为了把所有数据都翻出来，而是为了找到那个能开启地宫的**"钥匙"**。只有当视觉感应或逻辑感应足够强烈时，地宫里的全息真相才值得被重构。
