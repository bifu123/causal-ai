# 因果星空：事件视界与存在主义检索算法实现蓝图

## 1. 哲学设定：上帝视角 vs 观察者视角

在因果星空的设计理论中，存在两种截然不同但又相辅相成的视角：

*   **上帝视角（客观物理规律）**：
    *   由前端 3D 物理引擎和后端的全局代谢引擎（`metabolism.py`）维持。
    *   节点之间的语义相似度决定了它们在空间中的绝对距离（相似则聚，相异则散）。
    *   节点的全局权重随着时间的推移自然衰减，直至被所有人遗忘并封存入地宫。
*   **观察者视角（主观意志坍塌）**：
    *   由多租户权重表（`ains_user_weights`）和**事件视界（Event Horizon）**机制维持。
    *   当观察者（如张三）将意志锁定在某个节点（如“我想养猪”）时，宇宙为该观察者发生坍塌。
    *   只有与该意志相关的事件才会显现，无关的事件将在该观察者的视界中被“虚化湮灭”。

## 2. 核心概念：事件视界与 MAX_EYES

`MAX_EYES` 不是一个简单的节点数量硬截断，而是一个**“时空与语义的引力半径”**。

*   **精确语义距离公式**：
    利用 PostgreSQL 的 `pgvector` 插件计算余弦相似度，并将其映射为距离：
    `距离 D = (1 - 余弦相似度) * 100`
    *(例如：相似度 0.7 -> 距离 30)*
*   **视界判定**：
    *   **视界内 ($D \le MAX\_EYES$)**：节点与当前聚焦节点语义相近，或者属于直接的因果脉络。它们将被拉入视界，供 LLM 检索和前端高亮展示。
    *   **视界外 ($D > MAX\_EYES$)**：节点与当前意志毫不相干。它们将被排除在视界之外。

## 3. 落地执行计划

为了在不破坏“上帝视角”全局代谢规律的前提下，完美实现“观察者视角”的视界聚合，我们将采取以下轻量且优雅的改造方案：

### 3.1 环境变量配置
在 `.env` 文件中引入视界半径配置：
```env
MAX_EYES=30
```

### 3.2 后端接口改造 (`main.py` & `core/database.py`)
改造现有的节点聚焦接口（如 `search_causal_by_serial` 或点击节点的 API）。
**核心原则：惰性计算（查询时坍塌），绝对不在数据库中新增物理字段。**
1.  当用户或 Agent 聚焦某个节点时，除了返回该节点本身的数据和父子 ID 信息外，额外触发一次“视界扫描”。
2.  通过动态 SQL 瞬间计算全图活跃节点到该聚焦节点的语义距离 $D$：
    ```sql
    SELECT serial_id, node_id, parent_id, event_tuple,
           (1 - (semantic_vector <=> target_vector)) * 100 AS distance
    FROM ains_active_nodes
    WHERE (1 - (semantic_vector <=> target_vector)) * 100 <= {MAX_EYES}
    ORDER BY distance ASC;
    ```
3.  将查询出的**视界内节点**列表，作为一个名为 `event_horizon` 的 JSON 字段，动态打包进 API 的返回结果中。

### 3.3 前端视觉呈现 (`static/js/3d_main.js`)
实现"视界湮灭"与"上帝视角"的无缝切换：

#### 3.3.1 全局状态管理
*   `horizonNodes`（Set 类型）：存储当前事件视界内的节点 ID 集合。
*   `window.allNodeDistances`：从后端获取的全图节点距离映射表（`{ nodeId: distance }`）。
*   `window.currentMaxEyes`：当前生效的 `MAX_EYES` 值。
*   三者协同：当用户点击任意节点触发视界扫描后，后端返回 `event_horizon`、`all_distances`、`max_eyes` 三组数据，前端存储后调用 `updateHighlight()`，后者内部调用 `updateEventHorizonVisibility()` 完成可见度渲染。

#### 3.3.2 权重归一化函数 `getVisualWeight(weight)`
```javascript
function getVisualWeight(weight) {
    const w = Math.max(0, Math.min(1, parseFloat(weight || 0)));
    return w === 1.0 ? 0.3 : w;
}
```
*   将节点的 `survival_weight` 归一化到 [0, 1] 区间。
*   **特殊处理**：生存权重恰好为 1.0 的节点（所有人在同一时间共同关注时）会被降为 0.3，目的是防止它们在视觉上过于突兀（因为权重 1.0 的场景极为罕见，往往是瞬间的意志对齐，不应永久占据过高的视觉权重）。

#### 3.3.3 核心可见度函数 `updateEventHorizonVisibility()`
该函数在每次重绘（`updateHighlight`）时被调用，遍历场景中所有节点的 Three.js 3D 对象（球体 `sphere` 和精灵标签 `sprite`），根据节点是否在 `horizonNodes` 中，分别应用不同的透明度算法。

##### 3.3.3.1 上帝视角（无视界）
当 `horizonNodes` 为空时（用户未点击任何节点或点击了视角复位按钮），进入上帝视角：
```javascript
baseOpacity = 0.6 + weight * 0.35   // 范围：[0.6, 0.95]，上限 0.95
```
*   最低保底可见度为 0.6，即使权重为 0 的濒死节点也清晰可见。
*   最高可见度为 0.95（权重为 1.0 时），预留 0.05 的微弱透明空间以避免过于"死白"。
*   `action_tag === '又贞'` 的节点额外乘以 0.85，在视觉上温和削弱。

##### 3.3.3.2 视界内节点（观察者视角）——常数保底
节点在 `horizonNodes` 集合中时，所有节点统一使用常数保底可见度：
```javascript
horizonOpacity = 0.75   // 所有视界内节点整齐一致
```
*   **设计理念**：视界内节点的意义是"可见"，不应因权重差异而产生内部区分。0.75 的常量确保用户在观察者视角下能看到所有相关节点，且视觉整齐。（如果后续觉得视界内差异不够，可替换为激进函数如 `0.5 + weight × 0.45`）

##### 3.3.3.3 视界外节点（观察者视角）——纯距离线性衰减
节点不在 `horizonNodes` 集合中时，采用极简的纯距离线性衰减算法，**不涉及任何权重干扰**：

**第一步：获取语义距离**
```javascript
const maxEyes = window.currentMaxEyes || 30;
const distance = window.allNodeDistances[node.id] ?? (maxEyes * 2);
```
*   从后端返回的全局距离表中读取该节点与聚焦节点的语义距离 D。
*   若距离信息缺失，默认赋予 `maxEyes × 2`。

**第二步：纯距离线性衰减**
```javascript
if (distance > maxEyes) {
    const exceedRatio = (distance - maxEyes) / maxEyes;
    opacity = Math.max(0.05, 0.75 - exceedRatio * 0.7);
} else {
    opacity = 0.75;  // 距离恰好 ≤ MAX_EYES 的视界外边缘节点
}
```
*   **距离是唯一变量**：`exceedRatio` 直接决定可见度，从 0.75 线性滑落到 0.05（保底）。
*   **衰减效果**（以 maxEyes=30 为例）：

| 距离 D | exceedRatio | 可见度 |
|--------|-------------|--------|
| 30（视界边界） | 0 | 0.75 |
| 35 | 0.17 | 0.63 |
| 45 | 0.50 | 0.40 |
| 60（2 倍） | 1.00 | 0.05（触底） |
| 90+ | 2.00+ | 0.05（保底） |

**第三步：应用到 Three.js 对象**
```javascript
sphere.material.opacity = opacity;
sprite.material.opacity = opacity;
```

##### 3.3.3.4 算法总结
| 场景 | 可见度公式 | 范围 | 核心影响因素 |
|------|-----------|------|-------------|
| 上帝视角（无视界） | `0.6 + weight × 0.35` | [0.6, 0.95] | 仅权重 |
| 视界内节点 | `0.75`（常数） | 固定 0.75 | 无（统一保底） |
| 视界外节点 | `Math.max(0.05, 0.75 - exceedRatio × 0.7)` | [0.05, 0.75] | **纯距离**，越远越模糊 |


#### 3.3.4 上帝视角恢复
*   劫持 `btn-fit-view`（视角复位按钮）的点击事件。
*   执行 `horizonNodes.clear()` 清空视界集合，然后调用 `updateHighlight()`。
*   所有节点恢复到上帝视角的基础透明度，用户重新看到完整的客观因果星空。

#### 3.3.5 触发时机
*   **节点点击**（`onNodeClick`）→ 发送 `/api/v1/causal/click` 请求，带上 `need_all_distances=true` → 后端返回 `event_horizon`、`all_distances`、`max_eyes` → 前端存储 → 调用 `updateHighlight()` → 触发 `updateEventHorizonVisibility()`。
*   **搜索/跳转节点**（`jumpToNode`）→ 同上流程。
*   **视角复位**（`btn-fit-view`）→ 清空 `horizonNodes` → 恢复上帝视角。

## 4. 事件节点距离与代谢引擎同步的哲学思考和本系统的实现

### 4.1 哲学思考：时间是良药，遗忘把无关记忆推远

在人类的认知拓扑中，记忆并不是永恒不变的刻录，而是一个随时间不断代谢、风化、重塑的动态过程。

#### 4.1.1 记忆的模糊与距离的拉远
“时间是良药，都会过去的。”这句话在因果星空中有着精确的数学映射。当一个事件发生时，它充满了丰富的细节（全息文本），在我们的脑海中占据着核心位置。但随着时间的流逝，如果不去反复回想（缺乏因果连线和权重激活），这段记忆就会被“代谢”。
代谢的过程，就是细节被剥离、内容被提炼的过程。原本生动具体的记忆，变成了一个模糊的概括，甚至最终变成一个空壳。在这个过程中，这段记忆与其他事物的**关联性（相似度）必然会下降**，它在我们的心理空间中被推得越来越远。

#### 4.1.2 潜意识（向量）与表层意识（文本）的统一
在早期的设计中，我们为了保证检索的绝对精准，让节点的语义向量（潜意识）保持不变，只让文本内容（表层意识）随时间代谢。
但这导致了一个“身心不一”的认知错位：对于 Agent 来说，当它进入事件视界时，它会发现一些文本已经被代谢得面目全非的节点，依然凭借着最初全息状态下生成的锐利向量，死死地钉在视界中心。这违背了人类遗忘的自然规律。

真正的遗忘，不仅是表层文本的缩减，更是底层潜意识特征的泛化与丢失。

### 4.2 本系统的实现方案：第一性原理的胜利

为了完美模拟人类的遗忘机制，我们放弃了在距离公式中生硬地加入“时间惩罚常量”的工程捷径，而是选择回归第一性原理：**让向量同步跟随节点内容代谢而更新。**

#### 4.2.1 坚持纯粹的语义距离算法
我们保持事件视界的距离算法绝对纯粹，不受任何外部变量的直接干预：
`距离 D = (1 - 余弦相似度) * 100`
一个节点是否在视界内（`D <= MAX_EYES`），完全取决于它当前的语义向量与聚焦节点的语义向量之间的真实数学夹角。

#### 4.2.2 向量同步代谢（遗忘的推远）
当代谢引擎（`metabolism.py`）因为时间流逝和权重下降，将一个节点的详细内容提炼为简短摘要（甚至清空）时，系统会**同步为这个提炼后的新文本重新计算语义向量**。
由于新文本丢失了大量细节特征，新生成的向量在多维语义空间中会发生偏移，变得更加“泛化”。这种泛化直接导致它与当前聚焦节点的余弦相似度下降，计算出的距离 $D$ 变大。
当 $D$ 超过 `MAX_EYES` 时，这个节点就会自然而然地跌出事件视界。Agent 感受到的，将是一个真正随着时间流逝而变得模糊、疏离的记忆网络。

#### 4.2.3 地宫唤醒与锐化（记忆的拉回）
遗忘并非彻底的删除。当一个被推远的节点因为某种强烈的刺激被重新激活时，系统会从地宫（Necropolis）中恢复它的全息文本。
此时，系统会**再次同步为全息文本重新计算语义向量**。随着细节的回归，向量重新“锐化”，它在语义空间中的引力瞬间恢复。距离 $D$ 迅速缩小，这个节点又会被重新拉回事件视界，如同尘封的记忆被瞬间唤醒，历历在目。

---

**总结**：通过让向量字段与节点内容保持绝对的同步代谢，我们无需修改任何距离判定公式，就利用大语言模型 Embedding 本身的降维特性，完美实现了“时间遗忘把无关节点推远”的哲学构想。这不仅在视觉上照顾了关系，更在 Agent 的底层认知逻辑上实现了真正的拟人化。从而实现“因果是多意志主体相互影响下的多因多果的引力场”，用四维时空实现对它的事件物理映射。

*(注：关于距离是我们以后的升级实现，因为向量距离有时与关键字相关度不相符，我们后期会解决)*
